# FraudVault Deployment Guide

Single-server deployment using Docker Compose on Linux.

## Prerequisites

- Docker Engine 24+ with Docker Compose v2+
- 16 GB RAM minimum (32 GB recommended) -- Elasticsearch alone wants 2 GB
- 200 GB+ data partition (root partition alone is insufficient -- Docker data-root should point to the data partition)
- OpenSSL for TLS certificate generation
- Node.js 22.x (only needed for local TypeScript compilation during hot deploys)

## Required Environment Variables

All required variables use `${VAR:?error}` syntax in `docker-compose.yml` -- Compose will refuse to start if any are missing.

| Variable | Description |
|----------|-------------|
| POSTGRES_PASSWORD | PostgreSQL password (required, no default) |
| REDIS_PASSWORD | Redis password (required, no default) |
| MINIO_ACCESS_KEY | MinIO access key (required, no default) |
| MINIO_SECRET_KEY | MinIO secret key (required, no default) |
| JWT_SECRET | JWT signing secret (required, use 64+ char random string) |
| JWT_REFRESH_SECRET | JWT refresh token secret (required, different from JWT_SECRET) |
| INTERNAL_API_KEY | Inter-service authentication key (required) |
| ELASTICSEARCH_PASSWORD | Elasticsearch password (required) |

## Quick Start

### 1. Configure Environment

```bash
cp .env.example .env
# Edit .env -- generate random strings for all passwords and secrets
```

### 2. Generate TLS Certificates

```bash
bash nginx/generate-certs.sh
```

### 3. Build Sandbox Image

```bash
docker compose --profile build-only build scanboy-sandbox
```

### 4. Start All Services

```bash
docker compose up -d
```

This brings up all 17 services.

### 5. Verify

Wait approximately 2 minutes for all services to initialize, then:

```bash
curl -sk https://localhost/api/v1/health
```

### 6. Login

- Email: `admin@scanboy.local`
- Password: the configured password

Change the admin password immediately after first login.

## TLS Certificates

Self-signed certificates are generated by `nginx/generate-certs.sh` and stored in `nginx/certs/`.

For production, replace the self-signed files:

- `nginx/certs/scanboy.crt`
- `nginx/certs/scanboy.key`

## Hot Deploy (No Docker Rebuild)

Compile TypeScript locally, copy the built files into the running container, and restart the service. This avoids a full image rebuild.

```bash
# Ensure Node.js 22.x is on your PATH
npx tsc --project packages/<pkg>/tsconfig.json
docker cp packages/<pkg>/dist/. scan-boy-<service>-1:/app/packages/<pkg>/dist/
docker compose restart <service>
```

### Frontend Deploy

The frontend must be built from inside `packages/frontend` or the Tailwind content glob fails:

```bash
cd packages/frontend && npx vite build && cd ../..
docker compose exec frontend sh -c "rm -f /app/dist/assets/index-*.js /app/dist/assets/index-*.css"
docker cp packages/frontend/dist/. scan-boy-frontend-1:/app/dist/
docker compose restart frontend nginx
```

## `restart` vs `up -d`

| Command | Behavior |
|---------|----------|
| `docker compose restart <service>` | Preserves container state (installed packages, runtime files). Use for code deploys. |
| `docker compose up -d` | Recreates containers if Compose config changed. Picks up new `.env` values. Destroys runtime state. Must reinstall `docker-cli` in dynamic-analysis afterward. |


```bash
tar czf backup-src.tar.gz --exclude=node_modules --exclude=dist packages/ docs/ docker-compose.yml nginx/ .env
```

## Monitoring

- **Health check:** `curl -sk https://localhost/api/v1/health`
- **Service logs:** `docker compose logs -f <service>`
- **Disk usage:** `df -h`

## Troubleshooting

### Service won't start

```bash
docker compose logs <service>
docker compose exec postgres pg_isready -U scanboy
docker compose exec redis redis-cli ping
```

### Database migration issues

```bash
docker compose exec postgres psql -U scanboy -d scanboy \
  -f /docker-entrypoint-initdb.d/001_initial_schema.sql
```

### Reset all data

```bash
docker compose down -v
docker compose up -d
```
